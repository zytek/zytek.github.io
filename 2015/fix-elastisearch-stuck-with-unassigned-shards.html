Fix ElastiSearch stuck with unassigned shards

Date 04 Apr 2015 Tags devops / elasticsearch / tips

In case your ElasticSearch cluster is stuck with unassigned shards you might want to try rerouting them to some other node.

curl -XGET http://localhost:9200/_cat/shards | grep UNASSIGNED | awk '{print $1 " " $2}' | while read -r index shard; do

    echo "Rerouting shard $shard from index $index"
    curl -XPOST 'localhost:9200/_cluster/reroute' -d "{
        \"commands\" : [ {
              \"allocate\" : {
                  \"index\" : \"$index\",
                  \"shard\" : $shard,
                  \"node\" : \"NODE-NAME\"
              }
            }
        ]
    }"
    sleep 5
done

This should work for Elasticsearch 1.3.x and 1.4.x

For big indexes with many shards you might want to route them to different nodes. ElastiSearch will eventually distribute it evenly once they are initialized.
